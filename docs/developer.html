TCRBAC Developer Commands

Quick reference for a new developer building, testing, running examples, debugging in VS Code, and generating HTML documentation for the TCRBAC .NET Framework 4.8 solution.

What This Page Does

This is the full command-line path for a new developer. Run the commands from the repository root unless a step says otherwise.

The source code targets .NET Framework 4.8. The build tools are run by a .NET SDK selected by global.json, which currently asks for SDK 8.0.100 with rollForward set to latestFeature.

Install Prerequisites

  • Git for cloning the repository.
  • .NET SDK 8.x. This is the simplest path because it matches global.json.
  • .NET Framework 4.8 Developer Pack or targeting pack, required because the projects target net48.
  • Node.js and npm, required by the Allure report workflow.
  • DocFX, installed as a global .NET tool.
  • Repo-local .NET tools, restored from .config\dotnet-tools.json.
git --version
dotnet --list-sdks
dotnet --info
node --version
npm --version

dotnet tool update -g docfx
docfx --version

If only SDK 9.x is installed, either install an 8.x SDK, run this shell with $env:DOTNET_ROLL_FORWARD = "Major", or change global.json for the local checkout. Installing SDK 8.x is preferred because it matches the repository policy without a per-shell override.

Using An Installed .NET SDK 9.x

The SDK version is selected by global.json in the repository root. The default file pins the toolchain to SDK 8.0.100 and only rolls forward within .NET 8 feature bands.

If a developer only has SDK 9.x installed and does not want to install SDK 8.x, edit global.json locally and allow a major-version roll-forward:

{
  "sdk": {
    "version": "8.0.100",
    "rollForward": "latestMajor"
  }
}

Alternatively, pin the file to the installed SDK 9 baseline:

{
  "sdk": {
    "version": "9.0.100",
    "rollForward": "latestFeature"
  }
}

Treat this as a local environment change unless the team intentionally updates the repository standard. The projects still target net48; this only changes which .NET SDK runs restore, build, test, and DocFX tooling.

Full First-Time Setup

Use this sequence after cloning. Replace the clone URL with the real repository URL if needed.

git clone <repository-url> TCRBAC.NET
cd TCRBAC.NET

dotnet --list-sdks
dotnet nuget list source
npm install
dotnet tool restore
dotnet restore TCRBAC.NET.sln

dotnet restore downloads all NuGet dependencies, including log4net, xUnit, coverlet, ReportGenerator support packages, and the .NET Framework 4.8 reference assemblies package used by CLI builds.

Build Everything

This builds the library, tests, and example project through the solution.

dotnet clean TCRBAC.NET.sln
dotnet build TCRBAC.NET.sln

Build outputs are written under build. The library output is under build\TomcatUserRbacPort, examples under build\examples, and tests under build\tests.

Run All Tests And Reports

Use the project test runner rather than plain dotnet test. It runs the xUnit tests, writes TRX output, generates Allure, generates coverage, and copies the report pages into the documentation site.

powershell -NoProfile -ExecutionPolicy Bypass -File tests\run-tests.ps1 -Reset

Generated outputs:

  • tests\TestResults\unit-tests.trx
  • tests\AllureReport\index.html
  • tests\CoverageReport\index.html
  • site\tests\allure-report\index.html
  • site\tests\coverage-report\index.html

Run The Examples

The validator example reads examples\tomcat\tomcat-users.xml and checks a username/password pair.

dotnet run --project examples\TomcatUserValidator -- tomcat tomcat
dotnet run --project examples\TomcatUserValidator -- tomcat wrong-password
dotnet run --project examples\TomcatUserValidator -- both both

Expected exit codes are 0 for successful authentication, 1 for authentication failure, and 2 for invalid usage or missing sample data.

Build And Deploy The Site

Run these commands in this order. The Tomcat icon script must run after every DocFX build because DocFX rewrites the generated HTML.

docfx metadata docfx.json
docfx build docfx.json

docfx docs\docfx.json
docfx examples\docfx.json

powershell -NoProfile -ExecutionPolicy Bypass -File docs\add-tomcat-icons.ps1

The deployed local site folder is site. The standalone documentation outputs are docs\site and examples\site.

If the standalone docs or examples build reports access denied while overwriting api\toc.yml, close programs that may have the file open and rerun the same command from a normal PowerShell prompt.

Serve The Site

Start a local DocFX web server after the build and icon pass complete.

docfx serve site --port 8080

Open http://localhost:8080. Refresh the browser after rebuilding the site.

Use these pages to verify the full deployment:

  • http://localhost:8080/docs/developer.html
  • http://localhost:8080/tests/index.html
  • http://localhost:8080/tests/allure-report/index.html
  • http://localhost:8080/tests/coverage-report/index.html

One Paste Command Sequence

Run this from an already cloned repository root when all prerequisites are installed.

npm install
dotnet tool restore
dotnet restore TCRBAC.NET.sln
dotnet clean TCRBAC.NET.sln
dotnet build TCRBAC.NET.sln
powershell -NoProfile -ExecutionPolicy Bypass -File tests\run-tests.ps1 -Reset
dotnet run --project examples\TomcatUserValidator -- tomcat tomcat
dotnet run --project examples\TomcatUserValidator -- tomcat wrong-password
docfx metadata docfx.json
docfx build docfx.json
docfx docs\docfx.json
docfx examples\docfx.json
powershell -NoProfile -ExecutionPolicy Bypass -File docs\add-tomcat-icons.ps1
docfx serve site --port 8080

Optional Release Build

dotnet build TCRBAC.NET.sln --configuration Release
powershell -NoProfile -ExecutionPolicy Bypass -File tests\run-tests.ps1 -Configuration Release -Reset

Troubleshooting

DocFX command not found

dotnet tool update -g docfx

Only .NET SDK 9.x is installed

$env:DOTNET_ROLL_FORWARD = "Major"
dotnet --info

This override is for the current PowerShell session. Installing SDK 8.x is still the cleanest setup for this repository.

Tomcat icons disappeared

powershell -NoProfile -ExecutionPolicy Bypass -File docs\add-tomcat-icons.ps1

VS Code still shows stale C# errors

Developer: Reload Window
C#: Restart Language Server
