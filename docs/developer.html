TCRBAC Developer Commands

Quick reference for a new developer building, testing, running examples, debugging in VS Code, and generating HTML documentation for the TCRBAC .NET Framework 4.8 solution.

Prerequisites

  • .NET SDK 8.0 or newer. The repo uses global.json with SDK 8.0.100 and rollForward set to latestFeature.
  • .NET Framework 4.8 runtime for running the built net48 binaries on Windows.
  • DocFX installed globally for HTML documentation.
  • Optional Node.js and Mermaid CLI only if you need to regenerate Mermaid-authored diagrams.
dotnet --info
dotnet tool install -g docfx
docfx --version

npm install -g @mermaid-js/mermaid-cli
mmdc --version

The Mermaid commands are optional. The committed docs already use SVG files under docs\assets.

Fresh Checkout Setup

Run this full sequence from a new checkout. Replace the first two lines with the real clone URL and local folder if needed.

git clone <repository-url> TCRBAC.NET
cd TCRBAC.NET

dotnet --info
dotnet nuget list source
dotnet restore TCRBAC.NET.sln
dotnet build TCRBAC.NET.sln
dotnet test TCRBAC.NET.sln

dotnet run --project examples\TomcatUserValidator -- tomcat tomcat

docfx build
docfx docs\docfx.json
docfx examples\docfx.json
docfx serve site

dotnet restore downloads every project dependency from NuGet. That includes log4net 3.3.1, Microsoft.NETFramework.ReferenceAssemblies.net48, Microsoft.NET.Test.Sdk, xunit, xunit.runner.visualstudio, and coverlet.collector.

The repo-local NuGet.config clears machine-specific package sources and uses nuget.org, so a new developer does not need a checked-in DLL under lib.

After docfx serve site, open the printed local URL. The main generated site is expected at http://localhost:8080 when DocFX selects port 8080.

Core Build Workflow

Run these from the repository root: D:\TCRBAC.NET.

dotnet restore
dotnet clean
dotnet build
dotnet test

Build outputs are written under build\<ProjectName>\<Configuration>\net48 for the library, build\examples\<ProjectName>\<Configuration>\net48 for examples, and build\tests\<ProjectName>\<Configuration>\net48 for tests.

Shared configuration files live in conf. The source, test, and example projects copy conf\log4net.config into their outputs as conf\log4net.config.

The projects restore log4net from NuGet with PackageReference Include="log4net" Version="3.3.1"; do not commit a local log4net.dll.

Release Build

dotnet build --configuration Release
dotnet test --configuration Release

Project-Specific Commands

Library

dotnet build src\TomcatUserRbacPort.csproj

Tests

dotnet build tests\TomcatUserRbacPort.Tests\TomcatUserRbacPort.Tests.csproj
dotnet test tests\TomcatUserRbacPort.Tests\TomcatUserRbacPort.Tests.csproj

Validator Example

dotnet build examples\TomcatUserValidator\TomcatUserValidator.csproj
dotnet run --project examples\TomcatUserValidator -- tomcat tomcat
dotnet run --project examples\TomcatUserValidator -- tomcat wrong-password

Generate HTML Documentation

Main Documentation Site

docfx docs\docfx.json
docfx serve docs\site

The site includes API documentation, guides, and diagrams. Open the local URL printed by docfx serve.

Examples Documentation Site

docfx examples\docfx.json
docfx serve examples\site

VS Code Tasks

Open the Command Palette and run Tasks: Run Task, then choose:

  • Build Docs
  • Build Examples Docs

Diagrams

Current source UML diagram artifacts:

  • docs/diagrams/src-class-diagram.md
  • docs/assets/src-class-diagram.svg

If Mermaid CLI is installed and you have a .mmd input file, render SVGs with:

mmdc -i input.mmd -o output.svg

VS Code Debugging

Open Run and Debug, then choose one of these launch targets:

  • TomcatUserValidator: tomcat/tomcat
  • TomcatUserValidator: both/both
  • TomcatUserValidator: failed login
  • Debug Tests: RbacFlowTests

Build and test tasks are also available from Tasks: Run Task:

  • Build TomcatUserValidator
  • Build Tests
  • Run Tests
