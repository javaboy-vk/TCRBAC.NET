TCRBAC Developer Commands

Quick reference for a new developer building, testing, running examples, debugging in VS Code, and generating HTML documentation for the TCRBAC .NET Framework 4.8 solution.

Prerequisites

  • .NET SDK 8.0 or newer. The repo uses global.json with SDK 8.0.100 and rollForward set to latestFeature.
  • .NET Framework 4.8 runtime for running the built net48 binaries on Windows.
  • DocFX installed globally for HTML documentation.
  • Node.js and npm. The test workflow uses the repo-local Allure CLI package restored by npm install.
  • ReportGenerator restored as a repo-local .NET tool from .config\dotnet-tools.json.
  • Optional Mermaid CLI only if you need to regenerate Mermaid-authored diagrams.
dotnet --info
dotnet tool install -g docfx
docfx --version

node --version
npm --version
npm install
dotnet tool restore

npm install -g @mermaid-js/mermaid-cli
mmdc --version

The Mermaid commands are optional. The committed docs already use SVG files under docs\assets. Allure and ReportGenerator setup is local to the repository.

Fresh Checkout Setup

Run this full sequence from a new checkout. Replace the first two lines with the real clone URL and local folder if needed.

git clone <repository-url> TCRBAC.NET
cd TCRBAC.NET

dotnet --info
dotnet nuget list source
npm install
dotnet tool restore
dotnet restore TCRBAC.NET.sln
dotnet build TCRBAC.NET.sln
.\tests\run-tests.ps1 -Reset

dotnet run --project examples\TomcatUserValidator -- tomcat tomcat

docfx build
docfx docs\docfx.json
docfx examples\docfx.json

dotnet restore downloads every project dependency from NuGet. That includes log4net 3.3.1, Microsoft.NETFramework.ReferenceAssemblies.net48, Microsoft.NET.Test.Sdk, xunit, xunit.runner.visualstudio, and coverlet.collector.

npm install restores the Allure command-line dependency. dotnet tool restore restores ReportGenerator.

The repo-local NuGet.config clears machine-specific package sources and uses nuget.org, so a new developer does not need a checked-in DLL under lib.

These commands update generated files only. They do not start a server. If the documentation site is already running on port 8080, refresh http://localhost:8080. The top navigation includes Tests, Allure, and Coverage.

Core Build Workflow

Run these from the repository root: D:\TCRBAC.NET.

dotnet restore
dotnet clean
dotnet build
.\tests\run-tests.ps1 -Reset

Build outputs are written under build\<ProjectName>\<Configuration>\net48 for the library, build\examples\<ProjectName>\<Configuration>\net48 for examples, and build\tests\<ProjectName>\<Configuration>\net48 for tests.

Shared configuration files live in conf. The source, test, and example projects copy conf\log4net.config into their outputs as conf\log4net.config.

The projects restore log4net from NuGet with PackageReference Include="log4net" Version="3.3.1"; do not commit a local log4net.dll.

Release Build

dotnet build --configuration Release
.\tests\run-tests.ps1 -Configuration Release -Reset

Allure And Coverage Reports

Use the test runner instead of plain dotnet test when you want the visible reports to stay current.

.\tests\run-tests.ps1 -Reset

The runner resets old output, runs the xUnit suite, writes TRX results, converts them to Allure results, generates the Allure HTML report, collects Cobertura coverage, generates the ReportGenerator HTML coverage report, and copies both reports into the generated documentation site.

Open reports directly from the command line:

start .\tests\AllureReport\index.html
start .\tests\CoverageReport\index.html

Update the documentation site files without starting a server:

docfx build

If the documentation site is already running, refresh these pages:

  • http://localhost:8080/tests/index.html
  • http://localhost:8080/tests/allure-report/index.html
  • http://localhost:8080/tests/coverage-report/index.html

Generate HTML Documentation

Main Documentation Site

Build the site files without starting a server:

docfx build

This updates the generated site folder only. If the main local documentation site is already running on http://localhost:8080, refresh the browser to see the updated Tests, Allure, and Coverage links.

Standalone Developer Docs Site

docfx docs\docfx.json
docfx serve docs\site

Examples Documentation Site

docfx examples\docfx.json
docfx serve examples\site

DocFX Serve Port

docfx build does not use a port because it only writes files to the generated output folder. A port is used only when a developer manually starts a DocFX web server with docfx serve.

Set the port on the docfx serve command with --port or -p:

docfx serve site --port 8080
docfx serve site -p 8081

Use a different port only when the preferred port is already used by another process. The build-only workflow in this repository does not start or change any server.

  • TomcatUserValidator: tomcat/tomcat
